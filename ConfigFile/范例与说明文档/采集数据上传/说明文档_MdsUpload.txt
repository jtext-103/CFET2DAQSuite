//关于 {a,b,c,d} 形式参数的说明

//示例：{3,0,1,4}

//该参数用来自动生成有重复主体和变化数字的相同项，比如 pxi0001 - pxi0016
// a、b、c、d 原则上应为 uint 数，否则可能有 BUG

// a 为数字设定位数，0 表示不设特定位数，3 表示将 1 变为 001，如果 设为 3 但数超过 3 位则用原数表示

// b 为起始数

// c 为每次增加的步长，也就是数字每次加多少

// d 表示总共有多少个重复项

// ChannelConfigs 中 {a,b,c,d} 必须出现两次，否则报错，其中 d 参数以 SourceAiData 的为准

// {a,b,c,d} 中间不能有空格和其他不相关字符，但能出现在任意位置，只能出现一次

//需要注意的是，SourceAIData 中的通道可能会出现 000 的情况，手动上传时也需使用 000 形式，如果实际 AI 通道为 0 也不会有问题

可能遇到的问题：
1、{a,b,c,d} 中只能出现 { 或 , 或数字，不能有任何空格或其它字符
2、注意该有的 "" 和不该有的，比如 true 就没有 ""
3、注意斜杠的方向
